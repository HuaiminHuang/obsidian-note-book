# 4 Parting Words 结束语

*保持一致性 (BE CONSISTENT)*。

如果您正在编辑代码，花几分钟查看周围的代码并确定其风格。如果他们在索引变量名中使用 `_idx` 后缀，您也应该这样做。如果他们的注释周围有小哈希标记框，请让您的注释周围也有小哈希标记框。

拥有风格指南的重点是拥有一个通用的编码词汇表，以便人们可以专注于您在说什么，而不是您如何说。我们在这里提出全局风格规则，以便人们知道词汇表，但局部风格也很重要。如果您添加到文件中的代码看起来与周围的现有代码截然不同，这会在读者阅读时打乱他们的节奏。

然而，一致性是有限度的。它更重地适用于局部和全局风格未指定的选择。一致性通常不应用作为以旧风格做事的理由，而不考虑新风格的好处，或代码库随时间收敛到更新风格的趋势。
